# Internacionalização (i18n)

## Estado atual

- **Locale ativo:** `pt` (único em `astro.config.mjs` e `src/i18n/config.ts`)
- **URLs:** sem prefixo (`/c/rugs`, não `/pt/c/rugs`)
- **HTML:** `lang="pt"` em `BaseLayout`
- **Moeda:** EUR, formatação `pt-PT` (`formatPrice` em `src/i18n/index.ts`)

## Ficheiros

| Ficheiro | Função |
|----------|--------|
| `src/i18n/config.ts` | `locales`, `defaultLocale`, `isLocale()` |
| `src/i18n/types.ts` | Interface `UiDictionary` (contrato de chaves) |
| `src/i18n/pt.ts` | Traduções PT (UI + `products.{slug}`) |
| `src/i18n/index.ts` | `useTranslations`, `localizeProduct`, `getLocalizedProducts` |

## Duas camadas de texto

### 1. UI da loja (`UiDictionary`)

Navegação, botões, filtros, footer, mensagens de carrinho, etc.

**Adicionar string:**

1. Propriedade em `src/i18n/types.ts` → `UiDictionary`
2. Valor em `src/i18n/pt.ts`
3. No componente:

```astro
---
import { useTranslations } from '../i18n';
const t = useTranslations(Astro.currentLocale);
---
<button>{t.product.addToCart}</button>
```

### 2. Produtos (`products` em `pt.ts` + D1)

Por `slug`:

```ts
products: {
  'meu-novo-slug': {
    name: '…',
    description: '…',
    care: '…',
  },
}
```

`localizeProduct(base, locale)` faz merge de `ProductBase` (D1) + copy (i18n).

**Atributos traduzidos em runtime** via mapas em `pt.ts`:

- `t.materials[material]`
- `t.styles[style]`
- `t.rooms[room]`
- `t.colors[color]`

Chaves em BD permanecem em inglês (`wool`, `living-room`).

## Navegação

`src/data/navigation.ts` usa **chaves**, não labels:

```ts
{ key: 'rugs', href: '/c/rugs' }
// Render: t.nav[item.key]
```

## Adicionar locale `en` (exemplo)

1. `src/i18n/en.ts` — copiar estrutura de `pt.ts`
2. `src/i18n/index.ts`:

```ts
import en from './en';
const dictionaries = { pt, en };
```

3. `astro.config.mjs`:

```js
locales: ['pt', 'en'],
```

4. `src/i18n/config.ts`: `export const locales = ['pt', 'en']`

5. D1 — inserir linhas em `product_translations` para cada slug com `locale = 'en'`

6. (Opcional) `routing.prefixDefaultLocale: true` para `/en/...`

7. Regenerar seed ou migração `0003` com traduções EN

## API e locale

```
GET /api/products?locale=pt
GET /api/products/:slug?locale=pt
```

Default: `pt` se param inválido ou ausente.

## Fallback estático

Se `env.DB` não existir (ex.: contexto sem Worker), `hasDatabase()` é falso e `getLocalizedProducts` usa `src/data/products.ts` + `pt.ts`.

## Badges

Chaves: `sale` | `new` | `bestseller`  
Labels: `t.badges.sale`, etc.

Não traduzir chaves em URLs de filtro (`?badge=bestseller`).
